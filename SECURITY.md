# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please open a private report by contacting the repository owner directly rather than filing a public issue with exploit details.

When reporting, include:

- Affected file(s) and feature flow
- Steps to reproduce
- Impact assessment
- Suggested fix (if available)

## Scope

Potentially sensitive areas include:

- API key handling in options/storage
- Network calls to third-party APIs
- Any logic that could leak captured page content

## Response

We aim to acknowledge reports quickly and provide fix status updates as patches are prepared.
